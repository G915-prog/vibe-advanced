Lesson 1 of 4

What is React and why
does it matter to you?

You've already shipped a page with plain HTML and CSS. That works great for static content. But the moment you want things to change — a button that updates a score, a form that shows different fields based on input, a leaderboard that refreshes — you need a way to manage that changing state.

React is a JavaScript library that solves exactly this. Instead of manually finding elements in the DOM and updating them, you describe what the UI should look like for any given state, and React handles the updates. It's declarative, not imperative.

For vibe coders, React is the difference between asking AI to "add a button" and asking AI to "build me a full interactive app." The AI knows React deeply — it will scaffold components, wire up state, and handle edge cases you haven't thought of yet. Your job is to prompt it well.

The vibe coder's take: You don't need to memorise React's internals. You need to understand its concepts well enough to prompt precisely and debug intelligently. That's exactly what this module teaches.

Scaffolding with Vite

Vite is the fastest way to get a React app running. One command, and you have a fully working dev environment with hot reload. Here's what you'd run — but in this module, you'll prompt your AI to do it and explain each part.

bash # Scaffold a new React app with Vite npm create vite@latest my-app -- --template react cd my-app npm install npm run dev

This creates a project structure with src/ for your components, public/ for static assets, and a vite.config.js that handles everything. The dev server runs at localhost:5173 with instant hot reload.

Lesson 2 of 4

JSX, Components
and Props

JSX looks like HTML but it's JavaScript. It's how you describe UI in React. Every piece of your UI is a component — a function that takes some input (props) and returns some JSX to render.

jsx // A simple component — just a function that returns JSX function Greeting({ name }) { return ( <div className="greeting"> <h1>Hello, {name}!</h1> <p>Welcome to vibe coding.</p> </div> ); } // Using it — props are passed like HTML attributes function App() { return <Greeting name="JB" />; }

Key things to notice: component names are capitalised (React uses this to tell them apart from HTML tags). Props are destructured from the first argument. JSX uses className instead of class, and JavaScript expressions go inside curly braces { }.

Components can contain other components — that's how you build complex UIs from simple, reusable pieces. Your whole app is a tree of components, starting from a root App.

jsx // Composing components together function Card({ title, children }) { return ( <div className="card"> <h2>{title}</h2> <div className="card-body">{children}</div> </div> ); } function App() { return ( <Card title="My First Component"> <p>This content is passed as children.</p> </Card> ); }
Lesson 3 of 4

State with
useState

Props are data passed into a component. State is data that lives inside a component and can change over time. When state changes, React re-renders the component automatically — you never touch the DOM.

The useState hook gives you a state variable and a setter function. Call the setter and React re-renders with the new value.

jsx import { useState } from 'react'; function Counter() { // [currentValue, setterFunction] = useState(initialValue) const [count, setCount] = useState(0); return ( <div> <p>Count: {count}</p> <button onClick={() => setCount(count + 1)}> Increment </button> <button onClick={() => setCount(0)}> Reset </button> </div> ); }

This is the core loop of React: state changes → component re-renders → UI updates. You never write document.getElementById or innerHTML. You just update state and trust React to handle the rest.

Vibe coder tip: When prompting for state, describe what can change in your UI, not how to implement it. "I need a counter that tracks a number, increments on button click, and resets to zero" gives the AI everything it needs.

Side effects with useEffect

useEffect runs code after React renders. Use it for anything that reaches outside the component — fetching data, setting up subscriptions, updating the document title.

jsx import { useState, useEffect } from 'react'; function Counter() { const [count, setCount] = useState(0); // Runs after every render where count changes useEffect(() => { document.title = `Count: ${count}`; }, [count]); // ← dependency array: when to re-run // [] = run once on mount only useEffect(() => { console.log('Component mounted'); }, []); return <button onClick={() => setCount(c => c + 1)}>{count}</button>; }

Knowledge check

React Foundations Quiz

Five questions. Click an option to answer — you'll see the explanation immediately. No pressure, this is just for you.

Question 01 / 05
What is the main reason to use React over plain HTML for a dynamic app?
AReact loads faster than plain HTML in all cases
BReact lets you describe UI as a function of state, so the DOM updates automatically when data changes
CReact is required by Vercel and Cloudflare to deploy apps
DReact replaces CSS and removes the need for styling
Correct — B. React's core value is its declarative model: you describe what the UI should look like for any given state, and React figures out the minimal DOM updates needed. This makes complex, dynamic UIs far easier to reason about than manual DOM manipulation.
Question 02 / 05
In JSX, how do you write a CSS class on an element?
Aclass="my-class"
BclassName="my-class"
CcssClass="my-class"
Dstyle="class: my-class"
Correct — B. JSX is JavaScript, and class is a reserved keyword in JS. React uses className instead. Similarly, for becomes htmlFor on label elements.
Question 03 / 05
What does useState return?
AA single value that updates automatically
BAn object with get and set methods
CAn array with the current state value and a function to update it
DA Promise that resolves to the state value
Correct — C. const [count, setCount] = useState(0) — the array destructuring pattern gives you the current value and a setter. Calling the setter triggers a re-render with the new value.
Question 04 / 05
When should you use useEffect with an empty dependency array []?
AWhen you want the effect to run after every render
BWhen you want the effect to run only once, when the component first mounts
CWhen you want the effect to never run
DWhen the effect depends on all state variables
Correct — B. An empty [] tells React "this effect has no dependencies, so only run it once when the component mounts." This is the standard pattern for initial data fetching.
Question 05 / 05
What is the correct way to pass a value from a parent component to a child?
AStore it in localStorage and read it in the child
BUse a global variable declared outside both components
CPass it as a prop when rendering the child component
DCall the child component as a regular function
Correct — C. Props are React's way of passing data down the component tree. <Child value={myValue} /> — the child receives it as function Child({ value }). Data flows one way: parent → child.
